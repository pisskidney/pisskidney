Consider the searching problem:


Input: A sequence of n numbers $ A = \{a_1, a_2, ..., a_n\} $


Output: An index $i$ such that $v = A[i]$ or the special value $NIL$ if $v$ does not appear in $A$.


Write pseudocode for linear search, which scans throught the sequence, looking for $v$. Using a loop variant, prove that your algorithm is correct. Make sure your loop invariant fulfills the three necessary properties.

SEARCH(A, v)
for i:= 1 to A.length:
    if v == A[i]:
        return i
    return NIL

Loop invariant: The array $A[1, i - 1]$ does not contain the element $v$.


Initialization: When $i = 1$, the array $A[1, 0]$ is empty, therefore it doesn't contain the element $v$.


Maintenance: Since the algorithm returns $i$ when it finds $v$ on the $i$th index, if the algorithm reaches the $i+1$th iteration, we know that $v \notin A[1, i]$, maintaining the loop invariant.


Termination: The conditions for termination are: $v = A[i]$ or $i > A.length$. The first condition is pretty straight forward. If we terminate on $i > A.length$ it means we haven't found $v$ in $A[1, i-1]$. Since $i > A.length$ it means we have searched the entire array $A$, so the algorithm is correct.
